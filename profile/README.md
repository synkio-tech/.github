Get started
- Explore our repositories: https://github.com/synkio-tech
- Look for labels: good first issue, help wanted, documentation
- Read CONTRIBUTING.md and CODE_OF_CONDUCT.md in each repo before contributing

Contributing
We welcome contributions. Typical workflow:
1. Fork the repo and open a feature/bugfix branch
2. Run tests and linters locally
3. Open a pull request with a clear description and linked issue (if applicable)
4. Respond to review comments and iterate

Community & contact
- Open an issue in the relevant repository or start a discussion on the repo's Discussions tab
- Organization: https://github.com/synkio-tech

Project governance & policies
Each repo should include:
- LICENSE (recommended: MIT or Apache-2.0)
- CONTRIBUTING.md
- CODE_OF_CONDUCT.md
- SECURITY.md (how to report vulnerabilities)
